## 函数返回值

1、如果函数没有返回定义void，不能不写，c语言不支持
2、返回值类型一旦确定，如果return后面跟数据不对，C语言通常产生警告而不是错误，编译能通过。

强制返回会根据函数定义的类型返回出处理后的数据

```c
//情况1：基本类型不匹配
int func() {
    return 3.14; // 警告：double隐式转换为int，丢失精度
}
// 实际返回：3（截断小数部分）

//情况2：指针类型不匹配
int* func() {
    return "hello"; // 警告：指针类型不兼容
}
// 可能编译通过，但存在类型安全问题

//情况3：void函数返回值
void func() {
    return 5; // 错误：void函数不应返回值
}
```

```c++
//拓展一下C++部分  在C++中避免依赖隐式转换!
//情况1:基本类型不匹配
int func() {
    return 3.14; // 警告：double转换为int可能丢失数据
}
// 实际返回：3

//情况2：类类型不匹配
class A {};
class B {};

A func() {
    return B(); // 错误：无法将B转换为A
}

//严格的类型检查
const char* func() {
    return "hello"; // 正确：字符串字面量类型匹配
}

int* func2() {
    return "hello"; // 错误：无法将const char*转换为int*
}
```



## 函数声明

函数定义一般有两种方式
1、函数定义写在main函数之前，main直接调用，没有任务问题，
因为文档编译执行顺序就是正常
2、函数定义写在main函数之后，main直接调用，有问题，c原始语法是不支持这样的
因为文档编译执行顺序当执行到main函数，还没有编译到函数

要在最前面写一下函数声明
